## Project structure

The application follows a modular TypeScript architecture:

```
hostel management system/
├── index.html              # Main HTML entry point
├── tsconfig.json           # TypeScript configuration
├── styles/
│   └── style.css          # Application styles
├── src/                   # TypeScript source files
│   ├── app.ts            # Application entry point
│   ├── model/
│   │   ├── residents.ts  # Resident interface
│   │   └── rooms.ts      # Room interface
│   ├── services/
│   │   └── hostelService.ts  # Core business logic
│   ├── ui/
│   │   └── ui.ts         # User interface management
│   └── data/
│       └── roomsData.ts  # Initial room configuration
└── dist/                  # Compiled JavaScript (output)
    └── ... (mirrors src/ structure)
```

## TypeScript configuration

The project uses ES6 modules with strict type checking:

```json
{
  "compilerOptions": {
    "rootDir": "./src",
    "outDir": "./dist",
    "module": "es6",
    "target": "es6",
    "strict": true,
    "noEmitOnError": true,
    "moduleDetection": "force"
  }
}
```

<Info>
`noEmitOnError: true` prevents compilation if there are any TypeScript errors, ensuring type safety.
</Info>

## Installation steps

<Steps>
  <Step title="Install TypeScript">
    Install TypeScript globally or as a project dependency:
    
    ```bash
    # Global installation
    npm install -g typescript
    
    # Or as a dev dependency
    npm install --save-dev typescript
    ```
  </Step>
  
  <Step title="Navigate to project directory">
    ```bash
    cd "hostel management system"
    ```
  </Step>
  
  <Step title="Compile TypeScript files">
    Compile all TypeScript files to JavaScript:
    
    ```bash
    tsc
    ```
    
    This reads `tsconfig.json` and compiles all `.ts` files from `src/` to `dist/`.
  </Step>
  
  <Step title="Open in browser">
    Open `index.html` in your browser. The HTML file loads the compiled JavaScript:
    
    ```html
    <script type="module" src="./dist/app.js"></script>
    ```
  </Step>
</Steps>

## Development workflow

When modifying the source code, follow this workflow:

<Steps>
  <Step title="Edit TypeScript files">
    Make changes to any `.ts` file in the `src/` directory.
  </Step>
  
  <Step title="Compile changes">
    Run the TypeScript compiler:
    
    ```bash
    tsc
    ```
    
    Or use watch mode for automatic recompilation:
    
    ```bash
    tsc --watch
    ```
  </Step>
  
  <Step title="Refresh browser">
    Reload `index.html` in your browser to see the changes.
  </Step>
</Steps>

<Info>
Watch mode (`tsc --watch`) automatically recompiles files whenever you save changes, streamlining development.
</Info>

## Core components

### Application entry point

The application initializes in `src/app.ts`:

```typescript
import { hostelService } from "./services/hostelService.js";
import { UI } from "./ui/ui.js";

const data = new hostelService();
console.log("here", data);
new UI(data);
```

This creates the service layer and passes it to the UI layer for rendering.

## Customizing room configuration

To modify the available rooms, edit `src/data/roomsData.ts`:

```typescript
import { Rooms } from "../model/rooms"

export const roomsAvailability: Rooms[] = [
  { roomNumber: 101, isOccupied: false },
  { roomNumber: 102, isOccupied: false },
  { roomNumber: 103, isOccupied: false },
  // Add or remove rooms as needed
  { roomNumber: 110, isOccupied: false },
];
```

<Steps>
  <Step title="Modify roomsAvailability array">
    Add, remove, or renumber rooms in the array.
  </Step>
  
  <Step title="Compile TypeScript">
    ```bash
    tsc
    ```
  </Step>
  
  <Step title="Clear localStorage (if needed)">
    Open browser DevTools → Application → Local Storage → Clear to reset existing data.
  </Step>
</Steps>

<Note>
Changes to `roomsData.ts` only affect new installations. Existing data in localStorage will persist unless cleared.
</Note>

## Extending the data models

The system uses two interfaces that you can extend:

### Resident interface

```typescript
export interface Resident {
  id: string;
  name: string;
  age: number;
  phone: string;
  checkIndate: string;
  roomNumber: number;
}
```

To add fields (e.g., email, emergency contact):

<Steps>
  <Step title="Update the interface">
    Edit `src/model/residents.ts`:
    
    ```typescript
    export interface Resident {
      id: string;
      name: string;
      age: number;
      phone: string;
      email?: string;  // Optional field
      checkIndate: string;
      roomNumber: number;
    }
    ```
  </Step>
  
  <Step title="Update hostelService">
    Modify `addResident()` and `updateResident()` methods in `src/services/hostelService.ts` to handle the new field.
  </Step>
  
  <Step title="Update UI">
    Add corresponding form inputs and table columns in `index.html` and update `src/ui/ui.ts`.
  </Step>
  
  <Step title="Compile and test">
    ```bash
    tsc
    ```
  </Step>
</Steps>

### Room interface

```typescript
export interface Rooms {
  roomNumber: number;
  isOccupied: boolean;
}
```

You can extend this to include room types, capacity, or pricing:

```typescript
export interface Rooms {
  roomNumber: number;
  isOccupied: boolean;
  capacity?: number;      // e.g., single, double
  roomType?: string;      // e.g., "Standard", "Deluxe"
  monthlyRate?: number;   // pricing information
}
```

## Common issues

<AccordionGroup>
  <Accordion title="Changes not reflecting in browser">
    - Ensure you ran `tsc` after modifying TypeScript files
    - Hard refresh the browser (Ctrl+Shift+R or Cmd+Shift+R)
    - Check browser console for errors
  </Accordion>
  
  <Accordion title="TypeScript compilation errors">
    The `noEmitOnError: true` setting prevents compilation with errors. Fix all type errors before JavaScript files are generated.
  </Accordion>
  
  <Accordion title="Data not persisting">
    - Check if localStorage is enabled in your browser
    - Verify browser is not in private/incognito mode
    - Check DevTools → Application → Local Storage for data
  </Accordion>
  
  <Accordion title="No vacant rooms available">
    If all 10 rooms are occupied, delete a resident to free up a room, or add more rooms in `src/data/roomsData.ts`.
  </Accordion>
</AccordionGroup>
