  </Step>

  <Step title="Recompile TypeScript">
    ```bash
    tsc
    ```
  </Step>

  <Step title="Clear existing data (optional)">
    If you want to reset the system with new room data:

    1. Open browser DevTools (F12)
    2. Go to **Application** → **Local Storage**

    4. Refresh the page
  </Step>
</Steps>

<Note>
  New rooms only apply to fresh installations. Existing localStorage data takes precedence. Clear localStorage to see the new room configuration.
</Note>

### Using different room numbering schemes

You can use any numbering scheme:

<CodeGroup>
```typescript Sequential numbering
export const roomsAvailability : Rooms[] = [
    {roomNumber:1, isOccupied:false},
    {roomNumber:2, isOccupied:false},
    {roomNumber:3, isOccupied:false},
];
```

```typescript Floor-based (European style)
export const roomsAvailability : Rooms[] = [
    {roomNumber:001, isOccupied:false},  // Ground floor
    {roomNumber:101, isOccupied:false},  // First floor
    {roomNumber:201, isOccupied:false},  // Second floor
];
```

```typescript Letter prefixes
// Requires changing roomNumber type from number to string
export const roomsAvailability = [
    {roomNumber:"A101", isOccupied:false},
    {roomNumber:"A102", isOccupied:false},
    {roomNumber:"B101", isOccupied:false},
];
```
</CodeGroup>

<Warning>
  If using non-numeric room numbers, you must update the `Rooms` interface in `src/model/rooms.ts` to change `roomNumber: number` to `roomNumber: string`, and update the HTML input type accordingly.
</Warning>

## Data persistence customization

### Changing storage mechanism

By default, data is stored in `localStorage`. To use a different storage method:

<CodeGroup>
```typescript Session storage (clears on tab close)
// In hostelService.ts, replace localStorage with sessionStorage
saveData() {
  sessionStorage.setItem("rooms", JSON.stringify(this.rooms));
  sessionStorage.setItem("residents", JSON.stringify(this.resident));
}

loadData(): void {
  const storedRooms = sessionStorage.getItem("rooms");
  const storedResidents = sessionStorage.getItem("residents");
  // ... rest of method
}
```

```typescript Backend API (example)
// Replace localStorage calls with API requests
async saveData() {
  await fetch('/api/rooms', {
    method: 'PUT',
    headers: {'Content-Type': 'application/json'},
    body: JSON.stringify(this.rooms)
  });
  await fetch('/api/residents', {
    method: 'PUT',
    headers: {'Content-Type': 'application/json'},
    body: JSON.stringify(this.resident)
  });
}
```
</CodeGroup>

### Storage key customization

To use different localStorage keys (e.g., for multiple hostels):

```typescript src/services/hostelService.ts
private ROOMS_KEY = "hostel_downtown_rooms";
private RESIDENTS_KEY = "hostel_downtown_residents";

saveData() {
  localStorage.setItem(this.ROOMS_KEY, JSON.stringify(this.rooms));
  localStorage.setItem(this.RESIDENTS_KEY, JSON.stringify(this.resident));
}
```

