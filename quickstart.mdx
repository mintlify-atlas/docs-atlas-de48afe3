## Adding your first resident

<Steps>
  <Step title="Fill out the registration form">
    At the top of the page, you'll see a "Resident Registration" form with the following fields:

    - **Name**: Enter the resident's name
    - **Age**: Enter the resident's age
    - **Phone**: Enter a contact phone number
    - **Room Number**: Select an available room from the dropdown
    - **Check-In Date**: Select the check-in date

    <Info>
    The room dropdown automatically shows only vacant rooms, making it easy to avoid double-booking.
    </Info>
  </Step>

  <Step title="Submit the form">
    Click the **Add Resident** button to register the new resident.

    The system will:
    - Validate that the room is available
    - Create a unique ID for the resident
    - Mark the room as occupied
    - Save the data to localStorage
    - Display the resident in the table below
  </Step>

  <Step title="View the updated statistics">
    After adding a resident, check the "Room Overview" section to see updated statistics:

    - Total Rooms: 10
    - Occupied: 1 (shown in red)
    - Vacant: 9 (shown in green)
  </Step>
</Steps>

Here's the code that handles adding a resident from `src/services/hostelService.ts:44`:

```typescript
addResident(
  name: string,
  age: number,
  phone: string,
  roomNumber: number,
  checkInDate: string,
) {
  const room = this.rooms.find((r) => r.roomNumber === roomNumber);
  if (!room) {
    throw new Error("Room doesn't exist");
  } else if (room.isOccupied) {
    throw new Error("Room is already occupied");
  }

  const newResident: Resident = {
    id: Date.now().toString(),
    name: name,
    age: age,
    phone: phone,
    roomNumber: roomNumber,
    checkIndate: checkInDate,
  };
  this.resident.push(newResident);
  room.isOccupied = true;
  this.saveData();
}
```

## Editing a resident

To update a resident's information:

1. Click the **✏️ Edit** button next to any resident in the table
2. The form will populate with their current information
3. The submit button changes to "Update Resident Info" with an orange color
4. Make your changes (including transferring to a different room if needed)
5. Click **Update Resident Info**

<Warning>
When changing a resident's room, the system automatically frees the old room and marks the new room as occupied. The new room must be vacant.
</Warning>

## Removing a resident

To check out a resident:

1. Click the **Delete** button next to the resident in the table
2. Confirm the deletion in the dialog
3. The system automatically marks their room as vacant
4. The resident is removed from the table

## Searching for residents

Use the search bar above the resident table to find residents by name. The search is:

- **Case-insensitive**: Works with uppercase or lowercase
- **Real-time**: Results update as you type
- **Partial match**: Finds residents with names containing your search term


## Viewing statistics

The "Room Overview" section shows real-time statistics:

- **Total Rooms**: Total number of rooms in the system (10)
- **Occupied**: Number of rooms with residents (shown in red)
- **Vacant**: Number of available rooms (shown in green)

These statistics update automatically after any resident operation.

## Data persistence

All resident and room data is automatically saved to your browser's localStorage. This means:

- ✅ Data persists between browser sessions
- ✅ No server or database required
- ✅ Works offline
- ⚠️ Data is specific to your browser and device

<Tip>
To clear all data and start fresh, open your browser's DevTools (F12), go to the Application/Storage tab, find localStorage, and delete the `rooms` and `residents` entries.
</Tip>
