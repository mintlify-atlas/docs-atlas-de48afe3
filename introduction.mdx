## Welcome to Hostel Management System

A complete TypeScript-based solution for managing hostel residents and room availability. Built with vanilla TypeScript and localStorage for persistent data management, this system provides a clean, efficient interface for hostel administrators.

## System architecture

The application follows a clean TypeScript architecture with separation of concerns:

```typescript
src/
├── model/          # Data interfaces
│   ├── residents.ts
│   └── rooms.ts
├── services/       # Business logic
│   └── hostelService.ts
├── ui/             # User interface
│   └── ui.ts
├── data/           # Initial data
│   └── roomsData.ts
└── app.ts          # Application entry point
```

<Note>
All data is stored locally in your browser using localStorage with a 5MB capacity. No backend server is required.
</Note>

## Data models

The system uses two core interfaces:

**Resident interface:**
```typescript
interface Resident {
  id: string;
  name: string;
  age: number;
  phone: string;
  checkIndate: string;
  roomNumber: number;
}
```

**Room interface:**
```typescript
interface Rooms {
  roomNumber: number;
  isOccupied: boolean;
}
```

## Room configuration

The system comes pre-configured with 10 rooms (101-110). All rooms are initially vacant and ready for assignment:

```typescript
export const roomsAvailability: Rooms[] = [
  { roomNumber: 101, isOccupied: false },
  { roomNumber: 102, isOccupied: false },
  { roomNumber: 103, isOccupied: false },
  // ... rooms 104-109
  { roomNumber: 110, isOccupied: false },
];
```

<Info>
You can modify the room configuration in `src/data/roomsData.ts` to match your hostel's layout.
</Info>
